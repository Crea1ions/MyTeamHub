# 🧠 Orchestrator Vision

> Not an orchestration tool. Not an IDE replacement.  
> A cognitive system for thought, code, and memory.

---

## 🎯 WHAT IS Orchestrator?

Orchestrator is **a cognitive IDE** based on:
* **Files** (source of truth)
* **Agents** (perspectives)
* **Shared context** (explicit memory)

---

## ❌ WHAT IS Orchestrator NOT?

* ❌ NOT a SaaS Dashboard (no hosted backend)
* ❌ NOT a Visible Orchestrator (no workflow UI)
* ❌ NOT a LLM Pipeline (LLM is a service, not the system)
* ❌ NOT an Obsidian Clone (not a note-taking app)

---

## 🔥 CORE PRINCIPLES

### Principle 1: Single Responsibility
Each component has **ONE job** (Studio, Vault, Orchestrator, Dev Connector, OpenClaw)

### Principle 2: Explicit Memory
No hidden state. All memory in Vault (visible), all events logged (traceable)

### Principle 3: File-Based Truth
Source of truth is **files**, not databases (version controllable, human readable, distributed)

### Principle 4: Context Locality
At any moment, **exactly one context** is active

### Principle 5: No Intelligence Hiding
The system must be readable. Decisions visible, data flow traceable, no "AI magic"

---

## 🎮 AGENTS ARE PERSPECTIVES

9 agents available: Collaborateur, Explorateur, Analyste critique, Déconstructeur, Stress Tester, Codeur, Architecte, Documentaliste, Mentor

**Key Point**: Agents = **Modes** (user chooses which for each message)

---

## 📊 THE 3-LAYER THINKING MODEL

```
🧪 Ephemeral (Session) — Minutes to hours
   ↓ (when ready)
📦 Structural (Project) — Days to weeks
   ↓ (when stable)
📚 Foundational (Knowledge) — Months to years
```

---

## 🔄 THE FLOW

```
Start: Idea
  ↓
Session: Explore (agent-dev)
  ↓
Project: Formalize & execute (20-Projects)
  ↓
Knowledge: Stabilize & share (30-Knowledge)
  ↓
Reuse: Next project/session uses knowledge
```
