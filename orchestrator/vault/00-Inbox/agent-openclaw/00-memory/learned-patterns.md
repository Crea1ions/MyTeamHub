# 🔄 OpenClaw Learned Patterns

## Patterns Discovered

### Project Lifecycle
1. **Conception** → Idea in agent-dev/sessions
2. **Planning** → Formalized in agent-dev/projects-dev
3. **Execution** → Active work in projects-dev
4. **Completion** → Archived or integrated to Knowledge
5. **Reflection** → Insights extracted to 30-Knowledge

### Metadata Evolution
- **Status field** reflects actual state (draft → active → completed)
- **Tags** emerge organically; consolidate when patterns appear
- **Lien field** strengthens when cross-refs increase
- **Updated timestamp** shows true activity patterns

### Organizational Tendency
- Files cluster by **function** not **project**
- Knowledge centralizes **over time**
- Inbox acts as **temporary staging**
- MOCs become natural **convergence points**

---

## Emerging Rules

🔍 **Observation**: Files that lack lien fields are often orphaned
→ **Action**: Flag for connection review

🔍 **Observation**: Status never transitions backward (draft ≠ active again)
→ **Action**: Use archived for inactive, not draft

🔍 **Observation**: Tags stabilize after 3-4 uses
→ **Action**: Monitor for standardization opportunities
